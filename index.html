CHART API LIST

    D3.js

    특징

    1. 많은 데이터를 표현할 시 성능상에서 유리하다.
    2. 자주사용되는 기본 표현을 함수단위로 제공한다.
    3. 상당히 많은 양의 그래프를 표현할 수 있다.
    4. HTML의 SVG, CSS를 표현할 수 있다.
    5. 커스터마이징이 자유롭다.

    단점

    1. 구버전의 경우 최신 버전의 표현에 맞게 다시 바꿔주어야한다.
    2. 그래프 표현시, 별도의 작업을 해주어야한다.
    3. 안드로이드를 지원하지 않는다.

    Chart.js

    특징

    1. HTML5의 canvas를 기반으로 작성된 차트 라이브러리
    2. jQuery와 같은 다른 자바스크립트 프레임워크에 종속적이지 않음
    3. 차트의 각 구성요소들에 대한 많은 옵션을 제공
    4. 유연하며 디자이너, 개발자가 사용하기 간단하다.
    5. 반응형 라이브러리다.
    6. 안드로이드를 지원하는 라이브러리다.
    7. 8개의 차트타입을 그릴 수 있다.
    8. MIT라이선스 하에서 배포되므로, 라이선스 문구만 포함하면 소스 공개의무도 없고 수정, 재배포, 판매까지 가능한 완전히 무료

    단점

    1. 웹에서만 사용하는 것이라면, SVG를 사용하는 라이브러리 대비 성능이 좋지 않다.

    Flotr2

    특징

    1. HTML5의 canvas를 기반으로 작성된 차트 라이브러리
    2. jQuery와 같은 다른 자바스크립트 프레임워크에 종속적이지 않음
    3. 차트의 각 구성요소들에 대한 많은 옵션을 제공
    4. PlugIn 구조로 개발되어, 원본 수정없이 추가적인 기능 및 차트 추가 가능
    5. 대부분의 브라우저를 지원 (Internet Explorer 6.0이상 지원)
    6. 대부분의 모바일 브라우저 지원 (안드로이드, 아이폰의 브라우저 지원)
    7. 인쇄시에도 화면에 보이는 것과 동일하게 인쇄 가능
    8. MIT라이선스 하에서 배포되므로, 라이선스 문구만 포함하면 소스 공개의무도 없고 수정, 재배포, 판매까지 가능한 완전히 무료
    9. IE9, IE10 에서는 HTML5의 canvas 요소를 지원하기 때문에 flotr2.min.js 파일만으로 충분히 작동이 가능하다.

    단점

    1. IE8을 포함한 그 이하 버전에서는 flash를 대용으로 사용하도록 구성되어있다. 이를 테스트 하기 위해서는 웹서버에서 HTML파일을 로딩해야만 flash가 로딩되기 때문에 별도의 웹서버가 필요하다.
    2. 만일 IE8에서 테스트시 웹서버를 이용하지 않고 바로 로컬 파일 시스템에서 불러들이면 하얀 화면만 나올 것이다.
